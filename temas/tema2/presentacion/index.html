JavaScript y Consumo de APIs

Aplicaciones Web – Ing. Danny Páez

¿Qué es JavaScript?

  • Lenguaje de programación que se ejecuta en el navegador del usuario.
  • Permite crear interactividad: botones, formularios, efectos, etc.
  • Se utiliza junto con HTML y CSS para construir sitios web dinámicos.
  • También puede ejecutarse en el servidor (por ejemplo, con Node.js).

Variables en JavaScript

  • let: variable que puede cambiar su valor.
  • const: variable constante, no puede ser reasignada.
  • var: forma antigua, no recomendada en proyectos modernos.

      let nombre = "Danny";
      const edad = 30;
      var ciudad = "Quito";

Tipos de datos

  • string: cadenas de texto – "Hola"
  • number: números – 42
  • boolean: verdadero o falso – true, false
  • array: lista de elementos – [1, 2, 3]
  • object: conjunto de propiedades – { nombre: "Danny", edad: 30 }

Funciones en JavaScript

Las funciones agrupan código que se puede reutilizar.


      function saludar(nombre) {
        console.log("Hola " + nombre);
      }
      
      saludar("Danny");

También existen funciones flecha:


      const saludar = (nombre) => {
        console.log(`Hola ${nombre}`);
      };

Seleccionar elementos del DOM

  • document.getElementById(\"id\")
  • document.querySelector(\"selector\")

      const titulo = document.getElementById("miTitulo");
      const boton = document.querySelector(".btn");

Modificar elementos del DOM


      titulo.innerText = "Nuevo título";
      boton.style.backgroundColor = "blue";
      input.value = "Texto por defecto";

También puedes crear nuevos elementos con createElement y agregarlos al DOM.

Manejo de eventos

Con addEventListener puedes responder a eventos del usuario.


      const boton = document.getElementById("btn");
      
      boton.addEventListener("click", () => {
        alert("¡Hiciste clic!");
      });
  • Eventos comunes: click, submit, change, input

Resumen de 2.1 – Fundamentos JS

  • JavaScript permite crear interactividad en sitios web.
  • Se usa para manipular elementos HTML (DOM).
  • Variables, funciones y eventos son pilares esenciales.
  • Es la base para consumir APIs y construir aplicaciones dinámicas.

¿Qué es una API?

  • API significa Application Programming Interface.
  • Es un intermediario que permite que dos sistemas se comuniquen entre sí.
  • Permite enviar y recibir datos desde un servidor a una aplicación web.

Ejemplo: un catálogo que obtiene productos desde un servidor.

¿Qué es REST?

  • REST: Representational State Transfer
  • Es un estilo de arquitectura para diseñar servicios web.
  • Usa URLs para representar recursos (por ejemplo: /productos).
  • Interactúa con recursos usando métodos HTTP: GET, POST, PUT, DELETE.

Formato JSON

  • JSON = JavaScript Object Notation.
  • Es un formato ligero para intercambio de datos.
  • Se basa en pares clave-valor.

      {
        "nombre": "Danny",
        "edad": 30,
        "productos": ["Zapato", "Camisa"]
      }

JSON.parse() y JSON.stringify()

  • JSON.parse(): convierte texto JSON a objeto JavaScript.
  • JSON.stringify(): convierte objeto JavaScript a texto JSON.

      const texto = '{ "nombre": "Danny" }';
      const obj = JSON.parse(texto); // a objeto
      
      const nuevoTexto = JSON.stringify(obj); // a string otra vez

Métodos HTTP (REST)

  • GET: obtener datos (ej: ver productos)
  • POST: crear un nuevo recurso (ej: agregar producto)
  • PUT / PATCH: actualizar un recurso existente
  • DELETE: eliminar un recurso

Códigos de estado HTTP

  • 200 OK: Todo está bien
  • 201 Created: Recurso creado correctamente
  • 400 Bad Request: La solicitud está mal formada
  • 401 Unauthorized: El usuario no tiene permisos
  • 404 Not Found: El recurso no existe
  • 500 Internal Server Error: Fallo en el servidor

Resumen de 2.2 – APIs REST

  • Una API permite que el frontend y el backend se comuniquen.
  • REST organiza los recursos usando rutas y métodos HTTP.
  • Los datos se transmiten usando JSON.
  • Los códigos HTTP indican si la solicitud fue exitosa o fallida.

¿Qué es fetch()?

  • Es una función nativa de JavaScript para hacer solicitudes HTTP.
  • Permite conectarse con APIs externas o internas desde el navegador.
  • Devuelve una promesa que puede resolverse o fallar.

Ideal para obtener o enviar datos desde y hacia un servidor.

fetch() – Solicitud GET

Ejemplo: obtener productos desde una API pública.


      fetch('https://fakestoreapi.com/products')
        .then(response => response.json())
        .then(data => {
          console.log(data);
        })
        .catch(error => {
          console.error('Error:', error);
        });

fetch() – Solicitud POST

Ejemplo: enviar datos a una API.


      fetch('https://api.example.com/crear', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          nombre: 'Danny',
          producto: 'Laptop'
        })
      })
        .then(response => response.json())
        .then(data => {
          console.log('Éxito:', data);
        })
        .catch(error => {
          console.error('Error:', error);
        });

Manejo de errores con fetch()

  • Siempre usar .catch() para capturar errores.
  • Verificar si la respuesta fue exitosa con response.ok.

      fetch('https://api.example.com/data')
        .then(response => {
          if (!response.ok) {
            throw new Error('Error en la solicitud');
          }
          return response.json();
        })
        .then(data => console.log(data))
        .catch(error => console.error('Ocurrió un error:', error));

Mostrar datos en el DOM

Ejemplo: renderizar productos usando fetch y forEach.


      fetch('https://fakestoreapi.com/products')
        .then(res => res.json())
        .then(productos => {
          const lista = document.getElementById('productos');
          productos.forEach(producto => {
            const item = document.createElement('li');
            item.innerText = producto.title;
            lista.appendChild(item);
          });
        });

Resumen de 2.3 – Consumo con fetch()

  • fetch() permite conectar el frontend con APIs externas o internas.
  • Se usa principalmente con los métodos GET y POST.
  • Devuelve promesas que deben manejarse con .then() y .catch().
  • Es esencial para mostrar datos dinámicos en una aplicación web.

✅ Resumen – Tema 2: JavaScript y Consumo de APIs

  • JavaScript es el lenguaje que permite añadir interactividad a los sitios web.
  • Se ejecuta en el navegador del usuario y manipula el DOM con funciones, eventos y variables.
  • Una API REST permite intercambiar datos entre cliente y servidor usando rutas y métodos HTTP.
  • JSON es el formato más común para transmitir datos en APIs.
  • La función fetch() permite hacer solicitudes HTTP desde el frontend.
  • Con fetch() puedes obtener o enviar información, manejar respuestas y mostrar los datos dinámicamente.
  • Estos conceptos son la base para conectar el frontend con servicios reales.
